# service-worker

https://developer.mozilla.org/zh-CN/docs/Web/API/ServiceWorkerContainer/register

# register

  • 注册一个service-worker
  • scope参数：定义注册的作用域，该作用域下的文件能够被缓存（不一定对）
ServiceWorkerContainer.register(scriptURL, options)
    .then(
        function(ServiceWorkerRegistration) {
            // do something
        }
);

# ServiceWorkerRegistration

  • 注册后的sw实例
  • 可通过reg**.active.**postMessage('this message is from page')，向sw实例发送信息

# navigator**.**serviceWorker

  • 该方法可以接受来了sw实例的信息
navigator.serviceWorker.addEventListener('message', function(e) {
    console.log(e.data) // this message is from sw.js, to page
})

# MessageChannel

  • sw实例与页面通信也可以借助 MessageChannel对象

  • 实例化后会有port1和port2两个值

  • 用这两个值来相互通信

    // 接收
const messageChannel = new MessageChannel()
    messageChannel.port1.onmessage = e => {
    console.log(e.data) // this message is from sw.js, to page
}

    // 发送
reg.active.postMessage("this message is from page, to sw", [messageChannel.por2]);

# 注意

  1. 当用户第一次访问页面的时候，资源的请求是早于 Service Worker 的安装的，所以静态资源是无法缓存的；只有当 Service Worker 安装完毕，用户第二次访问页面的时候，这些资源才会被缓存起来；
  2. Cache Stroage 只能缓存静态资源，所以它只能缓存用户的 GET 请求；
  3. Cache Stroage 中的缓存不会过期，但是浏览器对它的大小是有限制的，所以需要我们定期进行清理；

# install

每次进入页面时都会执行install方法

  • waitUntil作用
  • 保证在install完成之前，把需要的文件都缓存下来
  • CacheStorage对象
  • 文档地址：https://developer.mozilla.org/zh-CN/docs/Web/API/CacheStorage
    • 用来缓存数据到浏览器
this.addEventListener('install', function(event) {
  // waitUntil作用：延长service-worker的寿命
  // 阻止浏览器在事件的异步操作完成之前终止该线程
  // 保证在install完成之前，把需要的文件都缓存下来
  console.log('install')

  event.waitUntil(
    // CacheStorage 对象
    // https://developer.mozilla.org/zh-CN/docs/Web/API/CacheStorage
    // 用来缓存数据到浏览器
    // caches.open打开一块缓存空间
    caches.open(swVersion).then(cache => {
      // 将需要的静态资源缓存起来
      return cache.addAll(['/src/a.js'])
    })
  )
})

# activate

只有当第二次进入页面时，才会触发activate方法

  • 该事件一般用来删除无用的caches
  • 项目每次打包发布会生成一个新的swVersion
  • install时，会将文件缓存进新的swVersion
  • 然后在activate事件中，将旧的swVersion的缓存删除
this.addEventListener('activate', event => {
  event.waitUntil(
    caches.keys().then(keys => {
      return Promise.all[keys.map(key => {
        if (![swVersion].includes(key)) {
          return caches.delete(key); // 删除不在白名单中的 Cache Stroage
        }
      })]
    })
  )
})

# fetch

监听浏览器的所有fetch请求，对已经缓存的资源使用本地缓存回复

  • 劫持用户发出的http请求
  • 将请求的url去caches中查找
  • 如果找到了，直接返回cahche中的数据
  • 如果没有找到则向服务器请求该资源
  • 然后将该资源缓存到caches
  • 最后将数据返回给用户
this.addEventListener('fetch', event => {
  // respondWith：劫持用户发出的 http 请求，并把一个 Promise 作为响应结果返回给用户
  const isNeedCache = /.js$|.css$/.test(event.request.url)
  event.respondWith(
    // 使用用户的请求对CacheStorage进行匹配
    caches.match(event.request).then(response => {
      //该fetch请求已经缓存
      // 匹配成功，则返回存储在缓存中的资源
      if (response) {
        return response
      }
      // 匹配失败，则向服务器请求资源返回给用户
      return fetch(event.request)
        .then(res => {
          if (isNeedCache) {
            // 对响应结果进行深拷贝
            const resClone = res.clone()
            // 使用 cache.put 方法把这些新的资源存储在缓存中
            caches.open(swVersion).then(cache => {
              cache.put(event.request, resClone)
            })
          }
          return res
        })
        .catch(err => {
          console.log(err)
        })
    })
  )
})

# work-box

谷歌团队推出的基于service-worker进行了二次封装的一个库

https://developers.google.com/web/tools/workbox

# workbox-routing

功能：根据规则拦截网络请求，将caches中的数据作为响应体返回

  • 会创建一个runtime的缓存空间，缓存空间大小使用workbox-expiration来控制

  • 接收两个参数

  • 第一个参数是路由匹配规则

  • 第二个参数是路由匹配中的callback

  • 通常回调处理会使用work-box提供的策略

    • workbox-strategies库是用来提供策略的

# workbox-strategies

提供workbox-routing匹配中路由后，的处理策略，相当于拦截fetch方法

  • CacheFirst：缓存优先策略，如果缓存中有直接取缓存中的值，没有再去请求
  • NetworkFirst：网络优先策略，尝试从网络获取最新响应。如果请求成功，它将把响应放入缓存中。如果网络未能返回响应，则将使用缓存的响应
  • NetworkOnly：只从网络请求
  • CacheOnly：只从缓存获取（不太常用）
  • StaleWhileRevalidate：同时从缓存和网络请求资源，使用缓存的资源，并将请求到的资源更新缓存

# workbox-expiration

对runtime缓存设置过期时间

  • maxEntries：可以设置缓存条数，最早缓存的数据将被删除
  • maxAgeSeconds：设置最长期间

# workbox-precaching

在触发install时间时，对资源进行预缓存，在触发activate时，将过期的缓存删除

  • 会创建一个precache缓存空间

  • 参数为具有url和revision 属性的对象数组

  • revision是根据文件内容自动生成的

  • 如果构建时文件名带有hash，则不需要revision

  • 默认会将不再预缓存列表中的缓存删除

  • cleanupOutdatedCaches：默认为true，添加一个activate事件侦听器，该事件侦听器将清除由早期版本的Workbox创建的过时的缓存

import {precacheAndRoute} from 'workbox-precaching';

precacheAndRoute([
  {url: '/index.html', revision: '383676' },
  {url: '/styles/app.0c9a31.css', revision: null},
  {url: '/scripts/app.0d5770.js', revision: null},
  // ... other entries ...
]);
